Tech Stack

  - Supabase (PostgreSQL + Auth) via @supabase/ssr 0.10.3
  - shadcn/ui + Tailwind CSS v4
  - Recharts 3.x + date-fns 4.x
  - Hosting target: Vercel

Breaking Changes Discovered (Next.js 16 vs 14)
  - middleware.ts is DEPRECATED → renamed to proxy.ts, function export renamed
    from `middleware` to `proxy`
  - params in pages/layouts are now Promises → must `await params` before use
  - cookies() from next/headers is async → must `await cookies()`
  - @supabase/ssr: get/set/remove cookie methods deprecated → must use
    getAll/setAll instead

Fixes Applied During Build
  1. TypeScript error in BusyHoursChart.tsx
     - Error: Recharts Tooltip `formatter` prop type mismatch — explicit `number`
       annotation on `v` was not assignable to `ValueType | undefined`
     - Fix: Removed explicit type annotation, let TypeScript infer from context
     - File: components/dashboard/BusyHoursChart.tsx line 25

  2. Next.js 16 middleware → proxy migration
     - Warning: "The 'middleware' file convention is deprecated"
     - Created proxy.ts with `export async function proxy(...)`
     - Deleted middleware.ts
     - File: proxy.ts (new), middleware.ts (deleted)

  3. Both middleware.ts and proxy.ts conflict error
     - Error appeared briefly while dev server was hot-reloading
     - Resolved automatically once middleware.ts was deleted and server recompiled

Business Logic Implemented
  - Quota check before insert: Trial (unlimited 30 days) / Free (30/month) / Pro
  - RLS enforced: all queries use auth.uid() via Supabase policies
  - Status flow: pending → confirmed → arrived / cancelled / no_show
  - Calendar click-to-create: passes ?date=&time= to new reservation form
  - CSV export: UTF-8 BOM for Thai characters in Excel, Pro/Trial gated

Supabase Setup Required (manual — run in SQL Editor)
  - CREATE TABLE profiles (...)  with RLS policy "own_profile"
  - CREATE TABLE reservations (...) with RLS policy "owner_access"
  - CREATE INDEX idx_reservations_owner_date ON reservations(owner_id, reservation_date)
  - See reserveq-project.md Section 3 for full SQL
